# 프로젝트 진행 상황 로깅 규칙

## 개요
모든 작업 계획, 진행 상황, 중요한 컨텍스트 정보를 `docs/progress-log.md`에 체계적으로 기록합니다.

## 로깅 규칙

### 1. 필수 로깅 시점
- **작업 시작 시**: 작업 계획 및 목표 기록
- **중요 결정 시**: 아키텍처, 기술 선택, 설계 결정
- **작업 완료 시**: 결과물 및 다음 단계 기록
- **문제 발생 시**: 이슈 및 해결 방법 기록
- **컨텍스트 변경 시**: 요구사항 변경, 방향성 수정

### 2. 로그 엔트리 형식
```markdown
### [YYYY-MM-DD HH:MM] 작업 제목
**작업**: 작업 내용 요약
- 구체적인 작업 항목 1
- 구체적인 작업 항목 2

**중요 컨텍스트**:
- 놓치면 안 되는 정보 1
- 놓치면 안 되는 정보 2

**결과/결정사항**:
- 주요 결과물
- 중요한 결정사항

**다음 단계**: 다음에 해야 할 작업
```

### 3. 타임스탬프 형식
- **날짜**: YYYY-MM-DD
- **시간**: HH:MM (24시간 형식)
- **예시**: [2025-01-10 17:30]

### 4. 순서 관리
- 시간순으로 상단에 최신 로그 추가
- 각 날짜별로 섹션 구분
- 작업 순서를 명확히 추적 가능하도록 구성

### 5. 필수 기록 항목

#### 작업 계획
- 전체 작업 로드맵
- 우선순위 및 Phase 구분
- 예상 소요 시간

#### 기술적 결정
- 라이브러리 선택 이유
- 아키텍처 설계 결정
- 데이터베이스 스키마 설계

#### 진행 상황
- 완료된 작업 항목
- 현재 진행 중인 작업
- 예정된 다음 작업

#### 중요 컨텍스트
- 요구사항 변경사항
- 성능 목표 및 제약사항
- 외부 도구 사용 (Gemini 피드백 등)

#### 문제 해결
- 발생한 문제 및 원인
- 해결 방법
- 예방 방안

### 6. 체크리스트 관리
- 전체 작업 진행률 추적
- Phase별 완료 상태 표시
- 미완료 작업 우선순위 관리

### 7. 주요 결정사항 섹션
- 프로젝트 전반에 영향을 미치는 결정사항
- 향후 참조가 필요한 설계 정보
- 성능 목표 및 품질 기준

## 사용 방법

### Claude Code 사용 시
1. 작업 시작 전: 현재 상황 확인 및 계획 기록
2. 작업 중: 중요한 결정사항 실시간 기록
3. 작업 완료 후: 결과물 및 다음 단계 기록

### 컨텍스트 유지
- 새로운 Claude Code 인스턴스 시작 시 progress-log.md 우선 확인
- 이전 작업 내용 및 결정사항 파악
- 현재 위치 및 다음 단계 명확히 인식

## 예시

```markdown
### [2025-01-10 17:45] Supabase 설정 완료
**작업**: Supabase 프로젝트 생성 및 환경 설정
- Supabase 프로젝트 생성
- 환경 변수 설정 (.env.local)
- 데이터베이스 스키마 생성

**중요 컨텍스트**:
- 프로젝트 URL: https://xxx.supabase.co
- RLS 정책 적용 필요
- 이미지 스토리지 버킷 'post-images' 생성 완료

**결과/결정사항**:
- posts, categories, tags, post_tags 테이블 생성
- 관리자 계정은 단일 사용자로 제한
- 이미지 업로드 크기 제한: 5MB

**다음 단계**: 인증 시스템 구현
```

이 규칙에 따라 모든 작업을 체계적으로 기록하여 프로젝트 진행 상황을 투명하게 관리합니다.
